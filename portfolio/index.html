Automated Detection of Diabetic Retinopathy Using Machine Learning

This project implements an automated system to classify retinal fundus images into Diabetic Retinopathy (DR) and No Diabetic Retinopathy (No_DR) using image processing and machine learning techniques. The approach applies grayscale conversion and resizing, followed by Histogram of Oriented Gradients (HOG) feature extraction. A Support Vector Machine (SVM) with an RBF kernel is trained on the extracted features to accurately detect diabetic retinopathy without using deep learning models.

Features: Preprocessing of retinal images (grayscale & resizing), HOG feature extraction for texture and edge analysis, Binary classification: DR vs No_DR, Support Vector Machine (SVM) with RBF kernel, Training, validation, and testing pipeline, Single image prediction support, Visualization of preprocessing steps and HOG features, Confusion matrix and performance evaluation.

Tech Stack: Programming Language: Python, Image Processing: OpenCV, Feature Extraction: scikit-image (HOG), Machine Learning: scikit-learn (SVM), Visualization: Matplotlib, Seaborn, Environment: Google Colab / Jupyter Notebook.

Booking Cancellation Prediction System

A machine learning system that predicts whether a hotel booking will be canceled, helping businesses reduce revenue loss.

Machine learning project focused on predicting customer behavior using real-world data.

The Problem: Hotels often lose money due to last-minute booking cancellations. Without prediction, they cannot take preventive action.

My Solution: I built a machine learning model that analyzes booking data and predicts the likelihood of cancellation in advance, enabling proactive decision-making.

Key Features:

  • Data cleaning and preprocessing of real-world booking data
  • Predictive machine learning model for cancellation likelihood
  • Performance evaluation using accuracy, precision, and recall metrics
  • Clear visualizations for insights and business decision-making
Results: The model achieved 92% accuracy with balanced precision and recall, successfully identifying both canceled and non-canceled bookings. This enables businesses to flag high-risk bookings early and take preventive action.

Real-World Use Cases:
  • Identify high-risk bookings for targeted follow-up
  • Reduce cancellation losses through early intervention
  • Improve overbooking strategies based on predictions
  • Support data-driven revenue planning decisions
Tools Used: Python, Pandas & NumPy, Scikit-learn, Matplotlib / Seaborn, Google Colab

Role: End-to-end development including data preparation, model training, evaluation, and visualization.
